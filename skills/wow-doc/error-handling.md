# 异常处理

在文档生成/更新过程中，遇到以下情况时暂停并提示用户：

## 文件系统异常

| 异常类型 | 处理 |
|----------|------|
| `docs/` 目录创建失败 | `⚠️ 无法创建 docs/ 目录，请检查文件系统权限。` |
| 文件写入失败 | `⚠️ 文件写入失败：<路径>。请检查磁盘空间或权限。` |
| 已有文件读取失败 | `⚠️ 无法读取 <路径>，将作为新文档创建。是否继续？` |

## 分类与命名异常

| 异常类型 | 处理 |
|----------|------|
| 无法判断文档类型 | 向用户展示候选类型，请求确认 |
| 内容跨多个类型且无明显主导 | 建议拆分为多个文档，请求确认 |
| 命名冲突且无法自动区分 | 展示已有同名文件内容摘要，请用户决定更新还是新建 |

## 处理原则

- 遇到异常后停止、报告、等待用户指示
- 不在用户不知情的情况下覆盖已有文档
- 用户确认后从失败步骤继续执行
